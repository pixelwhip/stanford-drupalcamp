*No Outside Code

Permitted beyond this point

As a front-end developer working with Drupal, I spend a good deal of time writing custom code to get the HTML, CSS and JS I want. So when we were first approached with a project that where a custom user experience was a high priority and needed to launch on was going to be built on the Stanford Sites platform, I was a little skeptical about how far we could push it. But as it turned out, there is a lot of power and flexibility built into Stanford Sites and what limitations we did run into we're related more to functionality that would require a custom module than custom code in the theme. In some ways this was actually freeing. It was a little relief from analysis paralysis. It forces you to be creative with the tools you have available and think inside the box. The purpose of a Drupal theme is to improve upon the site's user experience. Your SITE enables a user accomplish a certain goal--register for an event, communicate with their peers, or in most cases to access your content. The THEME, improves upon that experience. By making the navigation clearer, the text more readable and making sure the experience is consistent with related on and off line properties. From a technical point of view, the theme does this by modifying the default HTML, CSS and JS. And that's what we're talking about today.

B.E.A.N.

Blocks Entities Aren't Nodes

Block entities. Which means they are fieldable, and you can create custom view modes for them. This module does it give you direct control per say of blocks, but it does allow you to leverage the same Display Suite interface for blocks as you use for nodes.
What makes these handy in terms of theming is they can also have view modes. Those view modes can have their markup tweaked with display suite--just like nodes.
Beans are entities that behave like nodes in terms of being field able and having different content types. But also behave like blocks in terms of their placement.
What makes these handy in terms of theming is they can also have view modes. Those view modes can have their markup tweaked with display suite--just like nodes.
